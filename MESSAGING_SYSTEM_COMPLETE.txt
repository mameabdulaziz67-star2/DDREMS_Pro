📊 DATABASE CHANGES
═════════════════════════════════════════════════════════════════════════════

MESSAGES TABLE:
  • Added: is_group (BOOLEAN)
  • Modified: receiver_id (now nullable)
  • Added: idx_is_group index

MESSAGE_RECIPIENTS TABLE (NEW):
  • id (INT, PK)
  • message_id (INT, FK)
  • user_id (INT, FK)
  • is_read (BOOLEAN)
  • read_at (TIMESTAMP)
  • created_at (TIMESTAMP)
  • Unique constraint on (message_id, user_id)

🎯 DEPLOYMENT STEPS
═════════════════════════════════════════════════════════════════════════════

STEP 1: Backup Database
  mysqldump -u root -p ddrems > backup_$(date +%Y%m%d_%H%M%S).sql

STEP 3: Verify Migration
  mysql -u root -p ddrems

  SHOW TABLES LIKE 'message_recipients';

  npm start
