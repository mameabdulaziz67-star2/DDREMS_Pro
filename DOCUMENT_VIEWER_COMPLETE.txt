📋 HOW IT WORKS
═══════════════════════════════════════════════════════════════

COMPLETE WORKFLOW:

1. Property Admin logs in
   Email: propertyadmin@ddrems.com
   Password: admin123

2. Click "📄 Document Verification"

3. Enter Property ID and search

4. See list of documents

5. Click "🔑 Enter Key & Review" on any document

6. Enter access key in modal

7. Click "✓ Verify & View Document"

8. Document displays in full-screen modal with:
   - Property information at top
   - Document content in center
   - Document details at bottom
   - Verification buttons at bottom

9. Review the document

10. Take action:
    - ✅ Approve Property (makes it active)
    - ⏸️ Suspend Property (hides it)
    - ❌ Reject Property (deactivates it)
    - 🗑️ Delete Property (removes permanently)

11. Confirmation dialog appears

12. Action is executed

13. Success message shows

14. Modal closes and data refreshes

   Success: "Property rejected successfully!"

   Action: Permanently deletes property
   Effect: Property removed from database
   Confirmation: "PERMANENTLY DELETE? This cannot be undone!"
   Success: "Property deleted permanently."

💡 USAGE TIPS
═══════════════════════════════════════════════════════════════

1. Review Document First
   - Always review the document content
   - Check if it's legitimate
   - Verify it matches property details

2. Use Appropriate Action
   - Approve: Document is valid, property is good
   - Suspend: Need more information or temporary hold
   - Reject: Document is invalid or property has issues
   - Delete: Property should not exist (spam, duplicate)

3. Keyboard Shortcuts
   - Enter key in access key input = Verify
   - Esc key = Close modal (browser default)

4. Multiple Documents
   - Review all documents before deciding
   - Each document can be viewed separately
   - All use same verification buttons

🔧 TECHNICAL DETAILS
═══════════════════════════════════════════════════════════════

Component: DocumentViewerAdmin.js

Props:
  • propertyId (required) - Property ID to load documents
  • property (required) - Property object for display
  • userId (required) - Current user ID
  • onVerificationAction (optional) - Callback after action

State Management:
  • documents - Array of documents
  • loading - Loading state
  • showKeyModal - Access key modal visibility
  • selectedDoc - Currently selected document
  • accessKey - Entered access key
  • verifying - Verification loading state
  • viewedDocument - Currently viewed document
  • showDocumentModal - Document viewer modal visibility

API Calls:
  • GET /api/property-documents/property/:propertyId
  • POST /api/property-documents/verify-access
  • PUT /api/properties/:id/verify
  • DELETE /api/properties/:id
