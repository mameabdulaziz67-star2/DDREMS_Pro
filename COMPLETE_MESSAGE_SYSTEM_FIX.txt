================================================================================
DATABASE STRUCTURE
================================================================================

TABLES USED:
- messages: Stores all messages
  Columns: id, sender_id, receiver_id, subject, message, message_type, 
           is_read, is_group, created_at, updated_at

- message_recipients: Maps group messages to recipients
  Columns: id, message_id, user_id, is_read, read_at, created_at

- notifications: Stores user notifications
  Columns: id, user_id, title, message, type, is_read, link, created_at

- users: User information
  Columns: id, name, email, role, status, etc.

================================================================================
BACKEND ENDPOINTS
================================================================================

GET /api/messages/user/:userId
  - Fetch messages for a user
  - Role-based filtering
  - Returns: messages array with sender/receiver info

GET /api/messages/unread/:userId
  - Get unread message count
  - Returns: count, single_messages, group_messages

GET /api/messages/notifications/:userId
  - Get recent notifications
  - Returns: notifications array

POST /api/messages
  - Send single or group message
  - Body: receiver_id/receiver_ids, subject, message, message_type, is_group
  - Returns: message id and success status

POST /api/messages/bulk
  - Send bulk message to role
  - Body: receiver_ids/filter_role, subject, message, message_type
  - Returns: count of recipients

PUT /api/messages/:messageId
  - Edit message (sender only)
  - Body: subject, message
  - Returns: success status

DELETE /api/messages/:messageId
  - Delete message
  - Returns: success status

PUT /api/messages/read/:messageId
  - Mark message as read
  - Returns: success status

PUT /api/messages/read-all/:userId
  - Mark all messages as read
  - Returns: success status

================================================================================
TROUBLESHOOTING
================================================================================

If messages don't send:
1. Check backend is running on port 5000
2. Check database connection
3. Check browser console for errors (F12)
4. Check server logs for error messages

If messages don't appear:
1. Refresh the page
2. Check user role and permissions
3. Verify message was sent successfully
4. Check database for message records

If delete/edit doesn't work:
1. Check user permissions
2. Verify user is message sender (for edit)
3. Check browser console for errors
4. Restart backend server

If notifications don't update:
1. Check MessageNotificationWidget is integrated
2. Verify auto-refresh is working
3. Check browser console for errors
4. Restart backend server
