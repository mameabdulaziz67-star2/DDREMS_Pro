AGREEMENT MANAGEMENT - QUICK REFERENCE GUIDE
=============================================

ACCESSING THE FEATURE:
======================
1. Login to the system
2. Navigate to Agreement Management
3. View agreements in card format
4. Click "View Details" to see full information

OUTER VIEW (Agreement Card):
============================
Shows:
- Agreement ID with status badge
- Property title and location
- Customer name (ID: X)
- Owner name (ID: X)
- Creation date
- Action buttons based on role

Status Badges:
- ⏳ Pending (Orange)
- ✅ Accepted (Green)
- ❌ Rejected (Red)
- 🎉 Completed (Green)

INNER VIEW (Details Modal):
===========================
Shows:
1. AGREEMENT INFORMATION
   - Agreement ID
   - Status
   - Property details
   - Created date
   - Message

2. CUSTOMER INFORMATION
   - Full Name
   - User ID
   - Email
   - Phone
   - Address
   - ID Document
   - Occupation
   - Income

3. OWNER INFORMATION
   - Full Name
   - User ID
   - Email
   - Phone
   - Address
   - ID Document
   - Bank Account
   - Tax ID

4. PROPERTY INFORMATION
   - Title
   - Location
   - Type
   - Price (in ETB)

FILTER TABS:
============
📋 All - Show all agreements
⏳ Pending - Show pending agreements
✅ Accepted - Show accepted agreements
🎉 Completed - Show completed agreements

CUSTOMER ACTIONS:
=================
When Status = Pending:
- 💳 Submit Payment
- 📄 Upload Receipt
- 👁️ View Details

When Status = Accepted/Completed:
- 👁️ View Details

ADMIN ACTIONS:
==============
When Status = Pending:
- 📄 Generate Agreement
- 📧 Send Agreement
- 🔔 Send Notification
- 👁️ View Details

When Status = Accepted/Completed:
- 🔔 Send Notification
- 👁️ View Details

SEND AGREEMENT:
===============
1. Click "Send Agreement" button
2. Select recipient:
   - Customer - [Full Name]
   - Owner - [Full Name]
3. Click "Confirm"
4. Notification sent automatically

DATABASE TABLES USED:
====================
- agreement_requests (main data)
- properties (property info)
- users (name, email, phone)
- customer_profiles (customer details)
- owner_profiles (owner details)
- notifications (for sending notifications)

API ENDPOINTS:
==============
GET /api/agreement-requests/customer/:userId
GET /api/agreement-requests/admin/pending?admin_id=8
GET /api/profiles/customer/:userId
GET /api/profiles/owner/:userId
POST /api/agreement-requests/:id/send-agreement
POST /api/agreement-requests/:id/notify

TROUBLESHOOTING:
================
Issue: Profile not showing
Solution: Check if profile exists in database, system falls back to users table

Issue: Agreement not appearing
Solution: Check user role and permissions, verify agreement status

Issue: Send agreement not working
Solution: Verify recipient is selected, check notification settings

Issue: Data not updating
Solution: Refresh page, check database connection
