🎯 ISSUE FIXED
═════════════════════════════════════════════════════════════════════════════

ERROR: "Please select at least one recipient"

ROOT CAUSE:
  - When switching between send modes, the filter role wasn't being reset
  - This caused the checkbox list to show no users
  - Users couldn't select recipients even though they existed

SOLUTION APPLIED:
  1. Reset filterRole to 'all' when switching send modes
  2. Clear selectedUsers when switching modes
  3. Clear selectedUsers when changing filter role
  4. Fixed "Select All" button logic to handle empty lists

FILE: client/src/components/SendMessage.js

CHANGE 1: Reset state when switching modes
  - Single User mode: Reset filterRole and selectedUsers
  - Group Message mode: Reset filterRole and selectedUsers
  - Bulk (By Role) mode: Reset filterRole and selectedUsers

CHANGE 2: Clear selections when filter changes
  - When user changes role filter, clear selectedUsers
  - This prevents confusion with previously selected users

CHANGE 3: Fix "Select All" button logic
  - Check if filteredUsers.length > 0 before comparing
  - Prevents "Deselect All" showing when no users are available
