================================================================================
                          WORKFLOW ARCHITECTURE
================================================================================

CUSTOMER INITIATES REQUEST
  ↓
  Customer clicks "🔑 Request Access Key" or "🤝 Request Agreement"
  ↓
  Request sent to backend API
  ↓
  Backend stores in correct table:
    - request_key (for key requests)
    - agreement_requests (for agreement requests)
  ↓
  Property Admin receives notification

PROPERTY ADMIN REVIEWS & RESPONDS
  ↓
  Admin logs in and clicks "Agreement & Keys"
  ↓
  Sees merged list of pending requests
  ↓
  For KEY requests:
    - Clicks "Send Access Key"
    - Reviews key code in modal
    - Sends key to customer
  ↓
  For AGREEMENT requests:
    - Clicks "Forward to Owner"
    - Adds custom message
    - Forwards to owner

CUSTOMER RECEIVES RESPONSE
  ↓
  Customer receives notification
  ↓
  For KEY: Sees key code in dashboard
  For AGREEMENT: Sees status update
  ↓
  Can proceed with next steps

OWNER REVIEWS AGREEMENT (if forwarded)
  ↓
  Owner receives notification
  ↓
  Reviews forwarded agreement
  ↓
  Accepts or rejects
  ↓
  Customer receives final response

================================================================================
                            API ENDPOINTS
================================================================================

KEY REQUEST ENDPOINTS:
  POST   /api/key-requests                    - Create key request
  GET    /api/key-requests/customer/:userId   - Get customer's key requests
  GET    /api/key-requests/admin/pending      - Get pending key requests
  GET    /api/key-requests/admin/history      - Get key request history
  GET    /api/key-requests/:id/preview-key    - Preview key before sending
  PUT    /api/key-requests/:id/respond-key    - Respond to key request
  GET    /api/key-requests/broker/:brokerId   - Get broker's key requests

AGREEMENT REQUEST ENDPOINTS:
  POST   /api/agreement-requests                    - Create agreement request
  GET    /api/agreement-requests/customer/:userId   - Get customer's requests
  GET    /api/agreement-requests/admin/pending      - Get pending requests
  GET    /api/agreement-requests/admin/history      - Get request history
  PUT    /api/agreement-requests/:id/forward        - Forward to owner
  GET    /api/agreement-requests/owner/:ownerId     - Get owner's requests
  GET    /api/agreement-requests/broker/:brokerId   - Get broker's requests
  PUT    /api/agreement-requests/:id/respond        - Owner/broker responds

ALL ENDPOINTS NOW RETURN request_type FIELD ✅
